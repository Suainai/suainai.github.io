Java 基础 - 知识点

☕ Java 基础核心知识点

本文主要对 Java 基础知识点进行系统性总结与梳理。 @pdai

📦 数据类型

1. 包装类型

Java 拥有八个基本类型，每个基本类型都有对应的包装类型。基本类型与其对应的包装类型之间的赋值通过自动装箱与拆箱完成。

基本类型占据位数包装类型
boolean1Boolean
byte8Byte
char16Character
short16Short
int32Integer
float32Float
long64Long
double64Double

2. 缓存池 (Integer Cache)

💡 核心要点：new Integer(123) 每次都会新建一个对象；而 Integer.valueOf(123) 会优先使用缓存池中的对象。

基本类型对应的缓存池范围如下：

  • boolean：true 和 false
  • byte：所有值 (-128 ~ 127)
  • short：-128 ~ 127
  • int：-128 ~ 127
  • char：\u0000 到 \u007F (0 ~ 127)

🧵 String 深入解析

1. 概览

String 被声明为 final，因此不可被继承。
其内部使用 char 数组存储数据，该数组同样被声明为 final，这意味着 value 数组初始化之后就不能再引用其它数组，且 String 内部没有提供改变该数组的方法，从而保证了 String 的不可变性。

2. 不可变的好处 (Immutability)

  1. 🛡️ 安全性：String 经常作为网络连接、文件路径等参数，不可变性可以防止恶意修改。
  2. 🔒 线程安全：天生具备线程安全，可以在多线程中安全共享，无需同步。
  3. ⚡ 缓存 Hash 值：String 的 hash 值经常被使用（如 HashMap 的 key），不可变使得 hash 值只需计算一次即可缓存。
  4. ♻️ String Pool 的基础：只有不可变，才能安全地使用字符串常量池来节约内存。

3. String, StringBuffer 与 StringBuilder

  • 可变性：
    • String：不可变
    • StringBuffer & StringBuilder：可变
  • 线程安全：
    • String：线程安全
    • StringBuffer：线程安全（内部使用 synchronized 进行同步）
    • StringBuilder：非线程安全（但性能最高，单线程下首选）

4. String.intern() 与常量池

运行时常量池（Runtime Constant Pool）中包含了字符串常量池。

⚖️ 运算与传值

1. 参数传递 (Pass by Value)

⚠️ 重点：Java 的参数传递只有值传递，没有引用传递！

  • 当把对象作为参数传递时，传递的是对象地址的值。
  • 如果在方法中改变对象字段，原对象会改变（因为指向同一个内存地址）。
  • 如果在方法中将指针指向一个新对象，对原对象没有任何影响。

2. 隐式类型转换

对于 short s1 = 1;，由于字面量 1 是 int 类型，若直接相加 s1 = s1 + 1; 会报错（无法隐式向下转型）。

🧬 面向对象：继承与多态

1. 访问权限控制

设计良好的模块会隐藏所有的实现细节（封装），应当尽可能地使每个类或成员不被外界访问。

🧩 Object 通用方法

1. equals() 与 == 的区别

  • ==：对于基本类型，比较值；对于引用类型，比较内存地址。
  • equals()：Object 默认实现是比较地址。通常需要被重写以比较对象的逻辑内容是否相等。

2. hashCode()

⚠️ 黄金法则：在覆盖 equals() 方法时应当总是覆盖 hashCode() 方法！

等价的两个对象散列值一定相同，但散列值相同的两个对象不一定等价。如果不重写 hashCode()，将导致在使用 HashMap 或 HashSet 时出现严重 Bug。

3. 深拷贝与浅拷贝 (clone())

  • 💧 浅拷贝 (Shallow Copy)：拷贝对象和原始对象的引用类型引用同一个对象。
  • 🌊 深拷贝 (Deep Copy)：拷贝对象和原始对象的引用类型引用不同对象，互不干扰。

💡 《Effective Java》建议：最好不要去使用 clone()，可以使用拷贝构造函数或者拷贝工厂来替代。

🔑 核心关键字

1. final

  • 数据：基本类型不可变，引用类型不可改变引用指向（但对象内部状态可变）。
  • 方法：禁止子类重写。
  • 类：禁止被继承（如 String，Math）。

2. static

属于类级别，所有实例共享。

  1. 静态变量：内存中仅存一份，所有实例共享。
  2. 静态方法：不依赖于实例，方法内不能使用 this 或 super，只能访问其他静态成员。
  3. 初始化顺序：
    父类静态代码块 -> 子类静态代码块 -> 父类普通代码块/构造函数 -> 子类普通代码块/构造函数

🪞 反射 (Reflection)

反射允许在运行时获取类的内部信息（字段、方法、构造函数）并进行动态操作。

优点：

  • 极高的扩展性（Spring IOC / AOP 的核心）。
  • 适用于开发调试工具和类浏览器。

缺点：

  • 🐢 性能开销：反射操作通常慢于直接调用。
  • 🔓 安全性问题：可能破坏封装（通过 setAccessible(true) 访问私有成员）。

⚠️ 异常 (Exception)

Throwable 是所有异常的父类，分为：

  1. Error：JVM 无法处理的系统级错误（如 OutOfMemoryError）。
  2. Exception：
    • 受检异常 (Checked Exception)：必须在编译时处理（try-catch 或 throws）。
    • 非受检异常 (RuntimeException)：运行时错误（如 NullPointerException），通常由代码逻辑问题引起。
