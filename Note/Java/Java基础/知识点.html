☕ Java 基础核心知识点

本文不仅仅是对 Java 基础知识点的简单罗列，更结合了底层原理与实际工程案例进行深度剖析。打好基础，才能在后期的架构设计和并发编程中游刃有余。

📦 1. 数据类型与缓存机制

1.1 基本类型与包装类型

Java 拥有八个基本类型。在面向对象编程中，集合类（如 List<Integer>）只能存放对象，因此 Java 提供了对应的包装类型，并通过自动装箱与拆箱进行隐式转换。

long8 bytesLong0L
double8 bytesDouble0.0d

⚠️ 避坑指南：自动拆箱时如果包装类为 null，会抛出 NullPointerException！

🧵 2. String 深入解析与性能优化

2.1 String 为什么是不可变的？

String 的底层是一个被声明为 final 的 char[] 数组（Java 9 之后优化为了 byte[] 加上编码标志位 coder），并且没有暴露任何修改该数组内容的方法。

不可变带来的三大优势：

  1. 🛡️ 安全性：作为网络连接、文件路径等参数时，防止被外部恶意或意外修改。
  2. 🔒 绝对的线程安全：可以在多线程环境下自由共享，完全不需要加锁。
  3. ⚡ Hash 缓存：String 广泛用作 HashMap 的 key。由于不可变，其 hashCode 只需计算一次并缓存。

🧬 4. 面向对象：抽象类 vs 接口

随着 Java 版本的迭代，接口的能力越来越强，抽象类的使用场景在逐渐萎缩。

对比维度接口 (Interface)抽象类 (Abstract Class)
设计核心定义契约与行为规范 (LIKE-A 关系)提取共性，代码复用 (IS-A 关系)
状态/成员变量只能是 public static final (常量)可包含任意状态变量，保存对象状态
方法实现Java 8+ 支持 default 和 static 方法支持普通方法与抽象方法
多继承一个类可以实现多个接口一个类只能继承一个抽象类

工程案例：接口的 Default 方法如何拯救了 Java 集合框架？
在 Java 8 中，JDK 团队需要给所有的集合类加上 forEach 和 stream 方法。如果不使用 default 方法，所有实现了 List 接口的老代码全部都会编译报错！通过 default 关键字，接口实现了完美的向后兼容。

5.1 equals() 与 hashCode() 黄金法则

🔥 阿里开发规范强制要求： 如果重写了 equals() 方法，必须重写 hashCode() 方法！

为什么？
如果两个对象 equals 为 true，但 hashCode 不同，把它们放入 HashSet 或 HashMap 时，会被存储到不同的哈希槽中，导致集合里出现逻辑上的“重复元素”或产生内存泄漏（无法移除对象）。

🔑 6. 核心关键字

6.1 final 关键字的三大作用

  • 变量：修饰基本类型表示数值不可变；修饰引用类型表示引用（指针）不可变，但对象里面的内容可以变。
  • 方法：禁止子类重写（Override），保证方法逻辑的绝对稳定。
  • 类：彻底封杀继承链路。JDK 中的核心类库如 String, Math, Integer 全都是 final 的，防止被黑客覆写注入恶意代码。

static 属于类级别，其生命周期与类一致，常用于单例模式、工具类方法和全局常量。

⚠️ 8. 异常架构 (Exception) 与最佳实践

8.1 异常分类图鉴

  • Error：系统级崩溃，如 OutOfMemoryError（堆溢出）、StackOverflowError（栈溢出）。程序无法也无需捕获处理。
  • Exception：
    • 受检异常 (Checked Exception)：如 IOException。编译器强迫处理，但在现代架构中被认为是一种累赘，Spring 源码中几乎废弃了受检异常，全部转为 RuntimeException。
    • 非受检异常 (Runtime Exception)：如 NullPointerException、IllegalArgumentException。通常代表代码逻辑缺陷，可以通过提前校验 if(obj != null) 避免。

8.2 现代工程案例：定义全局统一业务异常

在真实的微服务或 Web 项目中，我们会抛出自定义的非受检业务异常，再由全局异常处理器 (@RestControllerAdvice) 拦截统一响应给前端，让业务代码极其整洁。
