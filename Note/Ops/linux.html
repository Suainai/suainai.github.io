Linux 运维指南

对于运维工程师而言，熟练掌握 Linux 基础命令是保障业务系统稳定的第一步。本指南详细介绍了核心命令及其在运维场景下的实战用法。

🔍 第二部分：系统监控与性能分析 (Diagnostic)

当基础任务无法满足需求时，需要借助 top 等工具深入分析系统瓶颈。

1. top 命令详解

top 是 Linux 下最常用的性能分析工具，能够实时显示系统中各个进程的资源占用状况，类似于 Windows 的任务管理器。

1.1 用法与参数

top [-] [d delay] [q] [c] [S] [s] [i] [n] [b]

参数释义
d改变显示的更新速度（秒）
c切换显示模式（仅执行档名称 vs 完整路径）
i不显示闲置 (idle) 或僵尸 (zombie) 进程
n更新指定次数后退出
b批次模式，常搭配 n 参数将结果输出到文件

1.2 输出内容深度解析

第一行：系统运行信息
top - 22:00:48 up 20:17, 1 user, load average: 0.00, 0.01, 0.05

  • 显示当前时间、系统运行时长、登录用户数。
  • load average: 每 5s 检查一次活跃进程数计算出的平均负载（1分钟、5分钟、15分钟）。一般当此值除以 CPU 核数大于 3~5 时，系统负载压力已很高。

第三行：CPU 运行信息

  • us (User): 用户态 CPU 占比。
  • sy (System): 内核态 CPU 占比。
  • id (Idle): 空闲 CPU 占比。
  • wa (Wait): CPU 等待磁盘 IO 的时间。当该值偏高时，通常意味着磁盘读写缓慢。

第六行：进程参数列

  • PR: 优先级。
  • NI: Nice 值，修正优先级。
  • VIRT: 虚拟内存使用量。
  • RES: 常驻内存使用量。
  • S: 进程状态 (S:休眠, R:运行, Z:僵尸, D:不可中断睡眠)。

1.3 交互操作技巧

  • 查看多核: 在 top 界面按 1，可监控每个逻辑 CPU 的情况。
  • 进程过滤: top -p <PID> 查看指定进程信息；top -H -p <PID> 查看指定进程的所有线程。
  • 排序切换:
    • P: 按 CPU 占用率排序 (%CPU)。
    • M: 按内存占用率排序 (%MEM)。
    • T: 按累计执行时间排序 (TIME+)。

💡 第三部分：性能调优建议 (Best Practices)

  • Tip 1: 养成良好的脚本注释习惯，便于后续维护。
  • Tip 2: 使用 Ctrl+r 快速检索历史命令，提升日常操作效率。
  • Tip 3: 在分析 IO 问题时，着重关注 top 输出中的 wa 指标。
