Angular

Angular 完整学习笔记

Angular 是一个用于构建大型 Web 应用的前端框架，以组件化、依赖注入、模块化、响应式编程和工程化能力著称。
这份笔记以现代 Angular 学习路径为主线，按照“基础认知 → 组件模板 → 服务与依赖注入 → 路由表单 → RxJS → 工程化与面试题”的顺序整理，适合作为系统学习与项目查阅手册。

1. Angular 是什么

1.1 Angular 的定位

Angular 是一个完整的前端应用框架，适合：

  • 企业级后台系统
  • 大型中后台平台
  • 多页面复杂业务系统
  • 强工程化、强规范团队项目

1.2 Angular 的特点

  • 基于 TypeScript
  • 组件化开发
  • 强依赖注入体系
  • 内置路由、表单、HTTP 能力
  • RxJS 响应式编程深度集成
  • 工程化与规范化程度高

1.3 Angular、React、Vue 的区别简述

  • Angular：更偏完整框架，官方内置能力多
  • Vue：更渐进式，更灵活，易上手
  • React：更偏视图库生态，需要自行组合方案

  • 管道（Pipe）
  • 服务（Service）
  • 依赖注入（DI）
  • 路由（Router）
  • 表单（Forms）
  • RxJS

2.2 Angular 项目常见结构

2.3 启动 Angular 应用的理解

16. 状态管理思路

16.1 小型项目

  • 局部状态放组件
  • 共享状态通过服务管理

16.2 中大型项目

常见方案：

16.3 状态设计原则

  • 局部状态就地管理
  • 跨页面共享状态集中管理
  • 不要把所有状态都塞成全局单例

  • 减少不必要订阅

17.2 trackBy

列表渲染时可减少重复重建。

17.3 OnPush 思想

在明确数据流的情况下，OnPush 能减少不必要检查，提高性能。

19. Angular 面试题精选

19.1 Angular 的核心特点是什么？

答：

  • 组件化
  • TypeScript 驱动
  • 依赖注入
  • RxJS 集成
  • 工程化能力强

19.2 组件和指令的区别？

答：

组件本质上也是一种特殊指令，但组件有模板，普通指令更多是给已有元素附加行为。

19.3 @Input 和 @Output 的作用？

答：

  • @Input：父传子
  • @Output：子通过事件向父通知

19.4 Service 的作用是什么？

答：

封装公共逻辑、数据请求、共享状态、业务逻辑等，提高复用与解耦能力。

19.5 Angular 为什么常配合 RxJS？

答：

因为 Angular 中很多异步能力天然就是基于 Observable 的，RxJS 有助于统一处理异步流。

19.6 模板驱动表单和响应式表单的区别？

答：

  • 模板驱动更简单
  • 响应式表单更适合复杂表单与动态校验

19.7 生命周期中常用哪个钩子发请求？

答：

通常是 ngOnInit。

19.8 路由守卫常见应用场景？

答：

  • 登录校验
  • 权限控制
  • 防止未授权访问页面

19.9 如何优化 Angular 列表渲染？

答：

  • 使用 trackBy
  • 减少不必要渲染
  • 大列表考虑分页或虚拟滚动

19.10 Angular 适合什么项目？

答：

更适合大型、复杂、工程化要求高、团队协作规范强的项目。

20. Angular 学习路线建议

20.1 初级阶段

  • TypeScript 基础
  • 组件与模板
  • 指令与数据绑定
  • 生命周期

20.2 中级阶段

  • 服务与依赖注入
  • 路由
  • 表单
  • HTTP 请求
  • RxJS 常用操作符

20.3 高级阶段

  • 状态管理
  • 性能优化
  • 架构分层
  • 大型项目规范
  • 测试与工程化

20.4 真正高级的 Angular 能力

  • 不只是会写组件
  • 而是能驾驭强工程化项目
  • 能设计模块边界与服务职责
  • 能处理复杂异步流与大型团队协作规范

21. 总结

Angular 的学习重点，不只是语法，而是理解它的工程思想：

  • 组件化
  • 依赖注入
  • 响应式编程
  • 大型项目架构能力

Angular 的真正优势，在于它非常适合复杂企业级项目和规范化团队协作。
