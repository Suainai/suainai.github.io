JavaScript

JavaScript 完整学习笔记

JavaScript 是 Web 前端的核心编程语言，用来处理页面交互、数据逻辑、异步请求、模块组织以及运行时行为。
这份笔记按照“基础语法 → 数据类型 → 函数对象 → 异步编程 → 浏览器环境 → 工程实践”的顺序整理，适合作为系统学习和长期查阅手册。

1. JavaScript 是什么

1.1 JavaScript 的作用

JavaScript 主要用于：

  • 操作页面内容和样式
  • 响应用户交互事件
  • 发送网络请求，与后端通信
  • 处理数据、计算逻辑、业务流程
  • 控制动画、表单校验、组件状态
  • 在浏览器、Node.js、桌面端、移动端等多环境运行

1.2 JavaScript 与 ECMAScript 的关系

  • ECMAScript：语言标准
  • JavaScript：ECMAScript 的一种实现
  • 浏览器中的 JavaScript = ECMAScript + DOM + BOM + Web API

1.3 JavaScript 的特点

  • 解释型 / 即时编译语言
  • 动态类型
  • 单线程事件循环模型
  • 基于原型的面向对象
  • 支持函数式编程
  • 支持异步编程

4.4 变量命名规范

  • 只能由字母、数字、下划线、$ 组成

39. 垃圾回收与内存管理

39.1 垃圾回收基本概念

JavaScript 会自动回收不再使用的内存。

常见策略：

  • 标记清除
  • 引用计数（历史概念）

39.2 常见内存泄漏原因

  • 全局变量滥用
  • 闭包引用不必要对象
  • 定时器未清除
  • 事件监听未移除
  • DOM 引用未释放
  • 缓存无限增长

39.3 优化建议

  • 用完清理定时器
  • 页面销毁时移除事件监听
  • 避免巨型对象长期驻留
  • 谨慎使用闭包保存大对象

42. JavaScript 编码规范建议

42.1 推荐实践

  • 优先使用 const，需要重新赋值时用 let
  • 少用 var
  • 优先用 ===
  • 使用早返回减少嵌套
  • 小函数、单一职责
  • 命名语义化
  • 避免魔法数字
  • 公共逻辑提取复用

42.2 错误处理

  • 预期错误要捕获
  • 不要静默吞错
  • 异步代码务必处理失败分支

42.3 可读性建议

  • 一眼能看懂比“写得炫”更重要
  • 减少过度链式调用
  • 复杂逻辑写注释说明意图

43. 学习路线建议

建议按下面顺序系统学习：

  1. 基础语法、变量、数据类型
  2. 运算符、流程控制、函数
  3. 作用域、闭包、this
  4. 对象、数组、原型、类
  5. Promise、async/await、事件循环

54. JavaScript 面试题精选

54.1 var、let、const 的区别？

答：

  • var 是函数作用域，有变量提升，可重复声明
  • let 是块级作用域，存在暂时性死区，不可重复声明
  • const 也是块级作用域，声明时必须初始化，不能重新赋值

54.2 什么是闭包？

答：

闭包是函数与其词法作用域的组合。函数可以访问定义时外层作用域的变量，即使外层函数已经执行结束。

54.3 什么是原型链？

答：

对象查找属性时，如果自身没有，就会沿着原型对象继续查找，直到 null，这条链条就是原型链。

54.4 this 的指向规则有哪些？

答：

  • 普通函数：默认调用决定
  • 对象方法：指向调用它的对象
  • 构造函数：指向新实例
  • call/apply/bind：显式绑定
  • 箭头函数：继承外层 this

54.5 == 和 === 的区别？

答：

  • == 会做隐式类型转换
  • === 不做类型转换，要求值和类型都相等
  • 项目中更推荐 ===

54.6 Promise 有哪些状态？

答：

  • pending
  • fulfilled
  • rejected

状态一旦改变就不可逆。

54.7 async/await 本质是什么？

答：

它是基于 Promise 的语法糖，让异步代码更接近同步写法。

54.8 事件循环是什么？

答：

JavaScript 主线程执行同步任务，异步任务完成后进入任务队列，事件循环不断把可执行任务推入调用栈执行。

54.9 宏任务和微任务的区别？

答：

当前同步代码执行完后，会先清空微任务队列，再执行下一个宏任务。

54.10 深拷贝和浅拷贝的区别？

答：

  • 浅拷贝只复制第一层引用
  • 深拷贝会递归复制所有层级内容

54.11 防抖和节流的区别？

答：

  • 防抖：停止触发后一段时间才执行
  • 节流：规定时间内最多执行一次

54.12 箭头函数和普通函数的区别？

答：

  • 箭头函数没有自己的 this
  • 不能作为构造函数
  • 没有 arguments
  • 更适合简洁函数与回调场景

54.13 new 关键字做了什么？

答：

大致过程：

  1. 创建新对象
  2. 让新对象原型指向构造函数的 prototype
  3. 以新对象为 this 执行构造函数
  4. 返回新对象（除非显式返回对象）

54.14 call、apply、bind 区别？

答：

  • call：立即调用，参数逐个传
  • apply：立即调用，参数数组传
  • bind：返回绑定后的新函数

54.15 为什么 typeof null === 'object'？

答：

这是 JavaScript 早期实现的历史遗留问题。
