CSS

CSS 完整学习笔记

CSS（Cascading Style Sheets，层叠样式表）用于控制网页的样式、布局、动画与响应式表现。
这份笔记按“基础 → 布局 → 视觉 → 动效 → 响应式 → 工程化”的顺序整理，适合作为系统复习与日常查阅手册。

1. CSS 是什么

1.1 CSS 的作用

CSS 用来描述 HTML 元素的视觉表现，主要负责：

  • 控制文字、颜色、背景、边框、阴影等外观
  • 控制元素尺寸、间距、位置、排列方式
  • 控制响应式布局与不同屏幕适配
  • 控制交互效果、过渡动画、关键帧动画
  • 提高页面结构与表现分离的可维护性

1.2 CSS 的核心思想

CSS 的核心在于以下几个概念：

  • 选择器：选中要设置样式的元素
  • 声明：给元素添加样式规则
  • 层叠：多个规则同时作用时如何决定最终结果
  • 继承：某些样式会从父元素传给子元素
  • 优先级：不同规则冲突时谁生效

5. 级联、继承与优先级

5.1 层叠（Cascade）

当多个规则作用于同一个元素时，CSS 会根据以下因素决定最终样式：

  • 来源
  • 优先级
  • 是否 !important
  • 书写顺序

5.2 继承（Inheritance）

有些属性会自动从父元素传给子元素。

常见可继承属性：

  • color
  • font-family
  • font-size
  • line-height
  • text-align

常见不可继承属性：

  • width
  • height
  • margin
  • padding
  • border
  • display
  • position

5.3 优先级规则

优先级从高到低通常是：

  • !important
  • 行内样式
  • ID 选择器
  • 类 / 属性 / 伪类
  • 元素 / 伪元素
  • 通配符 *

5.4 !important

不建议滥用，原因：

  • 难维护
  • 打破正常层叠关系
  • 后续覆盖困难

5.5 现代层管理：@layer

作用：让不同样式层有更稳定的覆盖关系。

15. 定位

15.1 position 的几种取值

  • static：默认值
  • relative：相对自身原位置定位
  • absolute：相对最近的已定位祖先元素定位
  • fixed：相对视口定位
  • sticky：粘性定位

15.2 相对定位 relative

特点：

  • 不脱离文档流
  • 保留原本占位
  • 常用于作为绝对定位子元素的参考容器

15.3 绝对定位 absolute

特点：

  • 脱离文档流
  • 相对最近的非 static 祖先元素定位

15.4 固定定位 fixed

特点：

  • 相对浏览器视口固定
  • 常用于悬浮按钮、吸顶栏

15.5 粘性定位 sticky

特点：

  • 在一定滚动范围内表现为相对定位
  • 达到阈值后像 fixed 一样“吸附”

注意：父元素的 overflow 可能影响 sticky 生效。

16. 层叠上下文与 z-index

16.1 z-index 是什么

用于控制元素在 z 轴方向上的层级。

16.2 常见形成层叠上下文的条件

  • 定位元素 + z-index
  • opacity < 1
  • transform
  • filter
  • perspective
  • isolation: isolate
  • position: fixed
  • 某些现代布局与渲染属性

16.3 常见问题

明明 z-index 很大却盖不住别的元素，通常是因为：

  • 不在同一个层叠上下文中
  • 父级已经形成新的层叠上下文
  • 元素没有建立正确的定位关系

29. 常见问题与踩坑

29.1 height: 100% 不生效

原因：父元素没有明确高度。

29.2 z-index 不生效

通常是：

  • 没有定位
  • 不在同一层叠上下文
  • 父元素层级影响

29.3 margin: auto 为什么不能垂直居中

它通常用于水平方向的块级元素居中，垂直居中更推荐 Flex / Grid。

29.4 行内元素设置宽高无效

如 span 默认是行内元素，通常不能直接控制宽高。

34. CSS 高级布局与格式化上下文

34.1 什么是格式化上下文

浏览器在排版元素时，会根据不同规则形成不同的格式化上下文（Formatting Context）。

常见概念：

  • BFC：块级格式化上下文
  • IFC：行内格式化上下文
  • FFC：Flex 格式化上下文
  • GFC：Grid 格式化上下文

34.2 BFC（Block Formatting Context）

BFC 是 CSS 中非常高频的面试与实战知识点。

常见触发方式

  • float 不为 none
  • position: absolute/fixed
  • display: inline-block
  • display: flow-root
  • overflow 不为 visible
  • Flex/Grid 子项或容器在某些场景下也会形成独立布局环境

BFC 的常见作用

  • 清除内部浮动影响
  • 阻止外边距折叠
  • 避免文字环绕浮动元素

34.3 IFC

行内格式化上下文主要用于行内元素排版，例如文本、span、a 等在一行中的布局计算。

34.4 FFC 与 GFC

  • FFC：Flex 布局上下文
  • GFC：Grid 布局上下文

使用现代布局时，理解这些上下文有助于排查对齐、尺寸与换行问题。

35. 浏览器渲染与 CSS 性能

35.1 浏览器渲染流程中的 CSS

浏览器大致会：

  1. 解析 HTML 形成 DOM
  2. 解析 CSS 形成 CSSOM
  3. 结合生成 Render Tree
  4. Layout
  5. Paint
  6. Composite

35.2 Reflow 与 Repaint

Reflow（回流 / 重排）

当元素尺寸、位置、布局结构变化时，浏览器需要重新计算布局。

例如：

  • 修改 width
  • 修改 height
  • 修改 display
  • 插入 / 删除 DOM

Repaint（重绘）

当元素外观变化但不影响布局时，只需重绘。

例如：

  • 修改 color
  • 修改 background
  • 修改 box-shadow

35.3 更推荐动画的属性

优先使用：

  • transform
  • opacity

因为它们通常更容易进入合成阶段，性能更好。

35.4 性能优化建议

  • 避免频繁修改布局属性
  • 减少复杂选择器
  • 避免过度嵌套
  • 用类切换代替多次逐条修改样式
  • 动画优先用 transform 与 opacity

38. CSS 面试题精选

38.1 盒模型是什么？

答：

盒模型由 content、padding、border、margin 组成。默认 box-sizing: content-box，项目中更常用 border-box。

38.2 display: none 和 visibility: hidden 的区别？

答：

  • display: none：元素不显示，也不占空间
  • visibility: hidden：元素不可见，但仍占据布局空间

38.3 em、rem、px 的区别？

答：

  • px：绝对像素单位
  • em：相对当前元素字体大小
  • rem：相对根元素字体大小

38.4 position 有哪些值？

答：

  • static
  • relative
  • absolute
  • fixed
  • sticky

38.5 absolute 相对谁定位？

答：

相对最近的非 static 祖先元素定位，如果没有，则相对初始包含块。

38.6 什么是 BFC？有什么作用？

答：

BFC 是块级格式化上下文，常见作用：

  • 清除浮动
  • 防止 margin 折叠
  • 阻止浮动元素影响外部布局

38.7 Flex 中 flex: 1 是什么意思？

答：

通常表示元素可伸缩并参与剩余空间分配，常可理解为 flex-grow: 1; flex-shrink: 1; flex-basis: 0%。

38.8 Grid 和 Flex 的区别？

答：

  • Flex 更适合一维布局
  • Grid 更适合二维布局
  • Flex 擅长一行或一列的排列分布
  • Grid 更适合复杂网格结构

38.9 为什么 z-index 不生效？

答：

常见原因：

  • 元素未建立正确定位上下文
  • 不在同一层叠上下文
  • 父元素已形成新的层叠上下文

答：

39. CSS 项目实践建议

39.1 初级阶段重点

  • 理解选择器、盒模型、文档流
  • 熟悉文本、背景、边框、间距
