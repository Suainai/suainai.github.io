Uni-App

uni-app 完整学习笔记

uni-app 是一个跨端应用开发框架，能够使用一套代码开发多个平台，如 H5、小程序、App 等。
这份笔记按照“基础认知 → 页面与组件 → 生命周期 → 网络与存储 → 跨端适配 → 发布构建 → 面试题”的顺序整理，适合作为系统学习与项目开发查阅手册。

1. uni-app 是什么

1.1 uni-app 的定位

uni-app 是一个跨端开发框架，适合：

  • 小程序项目
  • H5 项目
  • App 项目
  • 需要一套代码多端输出的业务场景

1.2 uni-app 的特点

  • 一套代码多端运行
  • 基于 Vue 语法体系
  • 提供统一跨端 API
  • 支持页面、组件、路由、存储、网络等能力
  • 适合快速开发多端业务

1.3 uni-app 常见应用场景

  • 企业内部业务小程序
  • 电商小程序
  • 多端活动页
  • 跨端信息展示项目
  • 轻量级 App

2. 项目结构基础

2.1 常见目录结构

2.2 关键文件说明

  • pages/：页面目录
  • components/：公共组件
  • static/：静态资源
  • pages.json：页面路由与窗口配置
  • manifest.json：应用配置
  • App.vue：应用入口组件

2.3 pages.json

主要用于配置：

  • 页面路径
  • 页面标题
  • 导航栏样式
  • tabBar
  • 分包配置

3. 页面与路由

3.1 页面注册

uni-app 页面通常通过 pages.json 配置。

3.2 页面跳转方式

常见 API：

  • uni.navigateTo
  • uni.redirectTo
  • uni.reLaunch
  • uni.switchTab
  • uni.navigateBack

3.3 常见区别

  • navigateTo：保留当前页，跳转到新页面
  • redirectTo：关闭当前页，跳转到新页面
  • reLaunch：关闭所有页面后打开目标页
  • switchTab：跳转 tabBar 页面
  • navigateBack：返回上一页

3.4 路由参数

页面中通过生命周期或路由参数获取。

13.1 下拉刷新

可在页面生命周期中处理：

  • onPullDownRefresh

13.2 上拉加载更多

可在：

  • onReachBottom

中处理分页加载。

13.3 长列表优化建议

  • 分页加载
  • 骨架屏
  • 避免一次性渲染超大数据
  • 使用虚拟列表思路（若业务复杂）

14. 权限与登录态处理

14.1 常见需求

  • 登录态校验
  • token 失效处理
  • 页面访问权限控制

14.2 常见处理思路

  • 启动时检查 token
  • 请求失败统一拦截 401
  • 无权限跳转登录页
  • 登录成功后回跳原页面

15. uni-app 与 Vue 的关系

15.1 共同点

  • 语法风格接近 Vue
  • 支持模板、数据、事件、组件化
  • 支持 props、methods、生命周期等思想

15.2 差异点

  • 标签体系不同，不是直接操作 DOM
  • 更强调跨端能力
  • 需要关注平台差异与条件编译
  • API 更偏小程序 / App 场景

16. 多端兼容与适配

16.1 兼容的核心难点

  • 不同平台标签与样式表现差异
  • 不同平台 API 权限差异
  • 页面路由和交互差异
  • 审核规则与发布流程差异

16.2 适配建议

  • 能统一就统一
  • 必要时使用条件编译
  • 不要依赖单平台特性过深
  • 做好多端测试

17. 性能优化建议

17.1 常见优化点

  • 图片压缩与懒加载
  • 长列表分页加载
  • 减少不必要渲染
  • 公共逻辑抽离复用
  • 请求层统一封装
  • 控制包体积

17.2 首屏优化思路

  • 减少首屏接口数量
  • 核心数据优先加载
  • 合理使用缓存
  • 延后不必要请求

18. 发布与构建

18.1 常见发布目标

  • H5
  • 微信小程序
  • 支付宝小程序
  • App

18.2 发布时要关注

  • 平台配置
  • 资源路径
  • 权限声明
  • 小程序审核要求
  • App 打包签名

18.3 构建前检查清单

  • 接口环境是否正确
  • 页面路径是否完整
  • 图片资源是否缺失
  • 条件编译逻辑是否正确
  • 登录与支付等核心流程是否已测试

19. uni-app 面试题精选

19.1 uni-app 的核心优势是什么？

答：

最大优势是一套代码多端运行，能够提高多端项目开发效率。

19.2 navigateTo 和 redirectTo 的区别？

答：

  • navigateTo：保留当前页，进入新页面
  • redirectTo：关闭当前页，跳到新页面

19.3 switchTab 适合什么场景？

答：

适用于跳转 tabBar 页面。

19.4 rpx 是什么？

答：

它是适合移动端适配的响应式单位，可根据屏幕宽度缩放。

19.5 uni-app 为什么需要条件编译？

答：

因为不同平台的 API、样式、能力存在差异，需要针对不同端输出不同代码。

19.6 页面常见生命周期有哪些？

答：

  • onLoad
  • onShow
  • onReady
  • onHide
  • onUnload
  • onReachBottom
  • onPullDownRefresh

19.7 uni-app 如何做本地缓存？

答：

通过 uni.setStorageSync、uni.getStorageSync 等 API。

19.8 uni-app 和普通 Vue 项目最大的差异是什么？

答：

uni-app 更关注跨端和平台能力，标签、API、适配方式都不同于传统浏览器 DOM 项目。

19.9 多端项目最容易踩的坑是什么？

答：

  • 平台兼容差异
  • 样式表现不一致
  • API 权限差异
  • 审核与发布差异

19.10 uni-app 适合什么项目？

答：

适合需要快速覆盖多个端、业务逻辑相对统一的项目。

20. uni-app 学习路线建议

20.1 初级阶段

  • 项目结构
  • 页面与路由
  • 基础组件
  • 数据绑定与事件
  • 生命周期

20.2 中级阶段

  • 网络请求封装
  • 本地存储
  • 组件化开发
  • 下拉刷新与分页加载
  • 多端样式适配

20.3 高级阶段

  • 条件编译
  • 登录态与权限体系
  • 跨端兼容处理
  • 性能优化
  • 发布与构建流程

20.4 真正高级的 uni-app 能力

  • 不只是会写页面
  • 而是能处理跨端差异
  • 设计稳定的业务组件
  • 管理复杂多端发布与兼容问题

21. 总结

uni-app 的核心价值，不只是“能跨端”，而是：

  • 提高开发效率
  • 降低多端维护成本
  • 让一套业务逻辑覆盖更多平台

但它的难点也很明显：

  • 平台差异
  • 兼容适配
  • 构建发布复杂度

uni-app 的初级能力是会写页面，中级能力是能做完整业务流程，高级能力是能处理复杂跨端兼容与工程化问题。
