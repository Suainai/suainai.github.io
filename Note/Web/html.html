HTML

HTML 完整学习笔记

HTML（HyperText Markup Language，超文本标记语言）用于描述网页的内容结构与语义。
这份笔记按照“基础结构 → 常用标签 → 表单表格 → 语义化 → SEO 与可访问性 → 最佳实践”的顺序整理，适合作为系统学习与长期查阅手册。

1. HTML 是什么

1.1 HTML 的作用

HTML 是网页的结构层，主要负责：

  • 描述页面内容与层级结构
  • 定义文本、图片、链接、表单、表格等元素
  • 提供语义信息，方便浏览器、搜索引擎、屏幕阅读器理解内容
  • 为 CSS 和 JavaScript 提供操作基础

1.2 HTML、CSS、JavaScript 的关系

三者在前端中分工明确：

  • HTML：负责结构
  • CSS：负责表现
  • JavaScript：负责行为

一句话理解：

HTML 搭骨架，CSS 做外观，JavaScript 赋予交互能力。

1.3 HTML 的特点

  • 使用标签描述内容
  • 标签可嵌套形成树状结构
  • 大多数标签由开始标签、内容、结束标签组成
  • 部分标签是空元素，没有结束标签
  • 支持语义化结构，利于维护与 SEO

16. SEO 基础

16.1 HTML 与 SEO 的关系

HTML 的结构和语义会直接影响搜索引擎理解页面内容。

16.2 SEO 相关建议

  • 合理使用 <h1> ~ <h6>

  • 图片带有合适的 alt

24. HTML 高级专题

24.1 浏览器解析 HTML 的基本过程

浏览器接收到 HTML 后，大致会经历：

  1. 解析 HTML，构建 DOM 树
  2. 解析 CSS，构建 CSSOM 树
  3. 合并生成渲染树（Render Tree）
  4. 进行布局（Layout / Reflow）
  5. 绘制（Paint）
  6. 合成（Composite）

这意味着：

  • HTML 结构会影响 DOM 构建效率
  • CSS 会影响首屏渲染
  • JavaScript 可能阻塞解析过程

24.2 script 对页面解析的影响

如果没有 defer 或 async，浏览器在解析到脚本时通常会：

  • 暂停 HTML 解析
  • 下载脚本
  • 执行脚本
  • 然后继续解析后续 HTML

因此常见优化方式：

  • 把脚本放在 body 结束前
  • 使用 defer
  • 对独立脚本使用 async

24.3 语义化对工程的价值

语义化不仅是“标签更好看”，更关系到：

  • SEO 排名理解
  • 屏幕阅读器朗读质量
  • 团队协作中的可读性
  • 后期维护与重构成本

25. HTML 与前端性能

25.1 影响性能的常见因素

  • 图片资源过大
  • 阻塞渲染的脚本
  • DOM 结构过深过复杂
  • 不合理的 iframe 嵌套
  • 首屏关键资源加载顺序不佳

25.2 HTML 层面的优化建议

  • 图片使用合适格式与尺寸
  • 使用懒加载
  • 关键 CSS 优先加载
  • 脚本使用 defer / async
  • 避免无意义标签嵌套
  • 保持 DOM 结构简洁

25.3 减少 CLS（布局偏移）

对于图片、广告位、视频位等动态内容，建议：

  • 明确宽高
  • 预留容器尺寸
  • 避免内容异步插入后整体抖动

26. HTML 与安全

26.1 XSS 基础认知

HTML 本身不执行逻辑，但若把不可信内容直接插入页面，容易产生 XSS 风险。

风险示例：

如果 userInput 来自不可信输入，就可能被插入恶意脚本。

26.2 安全建议

  • 不要随意使用 innerHTML
  • 不信任任何外部输入
  • 对内容做转义或过滤
  • 表单上传、URL 参数、富文本尤其要谨慎

26.3 iframe 安全

如果页面中使用 iframe，应关注：

  • 来源是否可信
  • 是否需要 sandbox
  • 是否存在 clickjacking 风险

27. HTML 工程化实践

27.1 组件化思维下的 HTML

在现代前端项目中，HTML 通常不再单独存在，而是：

  • 在 Vue / React / Angular 模板中编写
  • 在组件内部维护局部结构
  • 与样式和逻辑协作完成 UI

此时更重要的是：

  • 模板结构清晰
  • 语义明确
  • 便于拆分组件
  • 便于复用

27.2 团队协作建议

  • 页面骨架结构统一约定
  • 表单、表格、弹窗等用一致组件方案
  • 标题层级、ARIA 方案、埋点属性统一规范
  • 保持文档结构有一致风格

28. HTML 面试题精选

28.1 HTML 语义化有什么作用？

答：

HTML 语义化的作用主要有：

  • 提高代码可读性和可维护性
  • 有利于 SEO
  • 有利于无障碍访问
  • 更方便团队协作和后期重构

28.2 src 和 href 的区别是什么？

答：

  • src 用于引入资源本身，例如脚本、图片、音视频
  • href 用于建立当前文档和目标资源之间的关系，例如样式表、超链接

28.3 defer 和 async 的区别是什么？

答：

  • 两者都会异步下载脚本
  • async 下载完立即执行，顺序不保证
  • defer 会等 HTML 解析完成后按顺序执行
  • 多个有依赖关系的脚本更适合用 defer

28.4 alt 和 title 的区别？

答：

  • alt 是替代文本，图片加载失败或屏幕阅读器场景下非常重要
  • title 更多是补充提示信息，不等价于 alt

28.5 label 有什么作用？

答：

  • 提升表单可访问性
  • 点击文字也能聚焦或选中控件
  • 对屏幕阅读器用户更友好

28.6 常见的 HTML5 新特性有哪些？

答：

  • 语义化标签
  • audio / video
  • canvas
  • 新的表单类型与校验能力
  • localStorage / sessionStorage（更准确说是 Web API 生态能力）

28.7 meta viewport 有什么用？

答：

它用于控制移动端视口行为，使页面能按设备宽度正确布局与缩放。

28.8 为什么不建议用 div 代替按钮和链接？

答：

因为原生 button 和 a：

  • 自带语义
  • 自带键盘可访问性
  • 自带默认交互行为
  • 对辅助技术更友好

28.9 表单中 GET 和 POST 的区别？

答：

  • GET 参数在 URL 中，适合查询
  • POST 参数在请求体中，适合提交数据
  • GET 更适合幂等查询，POST 更适合创建/提交操作

28.10 为什么一个页面不建议有多个 h1？

答：

并不是绝对禁止，但通常一个页面应有明确主标题，多个 h1 容易让结构不清晰。实际项目要根据页面语义组织合理层级。

29. HTML 学习建议与项目实践

29.1 从初级到高级的学习路径

初级阶段

  • 熟悉常见标签
  • 能写出基础页面结构
  • 理解块级 / 行内元素
  • 会写表单、列表、表格

中级阶段

  • 掌握语义化结构
  • 理解 SEO 与可访问性
  • 会处理多媒体、响应式图片
  • 能写结构清晰、可维护的页面模板

高级阶段

  • 理解浏览器解析流程
  • 能从性能与无障碍角度设计结构
  • 能制定 HTML 规范
  • 能在组件化框架中设计稳定模板结构

29.2 项目中最重要的能力

  • 语义化思维
  • 结构规划能力
  • 表单设计能力
  • 可访问性意识
  • 与 CSS / JS / 框架协作的能力

30. 总结补充

HTML 看似简单，但真正写好并不只是会写标签，而是能写出：

  • 结构清晰的页面
  • 语义正确的模板
  • 对搜索引擎友好的内容组织
  • 对辅助技术友好的文档结构
  • 能和 CSS、JavaScript、框架工程化协作的代码

初级开发写得出页面，高级开发写得出结构合理、语义清晰、性能和可访问性都兼顾的页面。
