1. HTML 概念与文档结构

1.1 HTML 是什么

  • 超文本标记语言，用于描述网页结构
  • 由元素（Element）组成，元素包含标签与内容
  • 通过语义化结构让内容更易理解与维护

1.2 基础文档模板

1.3 元素、标签与属性

  • 元素：<p>内容</p>
  • 标签：起始标签 <p> 与结束标签 </p>
  • 属性：<img src="a.png" alt="描述">
  • 布尔属性：disabled、checked、readonly 等

1.4 块级与行内

  • 块级：div、p、h1-h6、ul、section
  • 行内：span、a、img、strong、em

1.5 注释

2. 文档元数据（head）

2.1 常用元数据标签

  • <title>：页面标题
  • <meta charset>：字符编码
  • <meta name="viewport">：移动端适配
  • <meta name="description">：搜索摘要
  • <link rel="stylesheet">：引入 CSS
  • <script>：引入 JS
  • <base>：基础 URL

11. 可访问性（A11y）

11.1 核心原则

  • 语义化结构优先

  • <label> 关联表单控件
  • 标题层级有序

11.2 ARIA

  • 仅在语义不够时使用
  • 常用：role、aria-label、aria-labelledby、aria-describedby

15. 最佳实践

  • 使用语义标签，避免无意义的 <div>
  • 保持结构清晰，缩进规范
  • 为可交互控件提供可访问文本
  • 使用 HTML 校验工具进行验证
  • 样式与脚本外置，减少内联

16. 常用标签速查

  • 链接/媒体：a、img、video、audio、source
  • 列表：ul、ol、li、dl、dt、dd
  • 表单：form、input、label、textarea、select、option、button
  • 表格：table、thead、tbody、tr、th、td
  • 语义区块：header、nav、main、section、article、aside、footer

18. 内容分类（Content Categories）

HTML 元素按内容模型可分为多类，理解这些有助于判断元素能否嵌套：

  • 元数据内容（Metadata）：base、link、meta、script、style、title
  • 流式内容（Flow）：正文中大多数元素都属于流式内容
  • 段落级内容（Phrasing）：可以出现在 <p> 内的文本级元素，如 span、a、em、img
  • 区块内容（Sectioning）：article、section、nav、aside
  • 标题内容（Heading）：h1-h6
  • 交互内容（Interactive）：a、button、input、select、textarea、details、summary
  • 嵌入内容（Embedded）：img、video、audio、iframe、canvas、svg

19. 全局属性详解

所有元素都可使用的常见属性：

  • id：唯一标识，配合 #id、label for、aria-labelledby
  • class：类名，供 CSS/JS 使用
  • title：提示文本（不应替代可访问性文本）
  • lang：语言声明（如 zh-CN）
  • dir：文字方向（ltr、rtl、auto）
  • tabindex：键盘焦点顺序（0、-1、正整数）
  • hidden：隐藏元素（辅助技术也隐藏）
  • contenteditable：可编辑内容
  • draggable：拖拽行为
  • spellcheck：拼写检查
  • translate：是否允许翻译
  • data-*：自定义数据属性

20. 常见元素详解

20.1 文本相关

  • strong 表示重要性，b 仅加粗外观
  • em 表示语气强调，i 仅倾斜外观
  • mark 用于高亮关注内容
  • small 用于注释、免责声明等次要内容
  • time 表示日期时间，可带 datetime

20.2 分组与布局

  • div：无语义容器
  • span：行内容器
  • figure + figcaption：自包含内容与标题
  • main：页面主体（通常唯一）

20.3 交互与披露

  • details + summary：可折叠内容
  • dialog：对话框（结合 open 属性）

20.4 代码相关

  • code：代码片段
  • pre：保留空格与换行
  • kbd：键盘输入
  • samp：程序输出
  • var：变量名

21. 表单详解

21.1 <form> 常用属性

  • action：提交地址
  • method：get / post
  • enctype：application/x-www-form-urlencoded、multipart/form-data
  • target：提交目标（如 _blank）
  • autocomplete：表单级自动完成
  • novalidate：禁用浏览器校验

21.2 <input> 通用属性

  • name、value、placeholder
  • required、readonly、disabled
  • min、max、step
  • pattern、maxlength、minlength
  • autocomplete、autofocus
  • list：关联 datalist
  • accept：文件类型限制（file）

21.3 输入类型补充说明

  • email：带基本格式校验
  • url：URL 格式校验
  • number：数字输入（仍需校验）
  • range：滑块
  • file：文件上传
  • date / time：原生日期/时间选择
  • checkbox：多选
  • radio：单选（同名为一组）
  • hidden：隐藏字段
  • button / submit / reset：按钮行为

21.4 关联元素

  • label：提高可访问性与点击范围
  • fieldset/legend：表单分组语义
  • select/option/optgroup：下拉选项
  • textarea：多行文本

22. 多媒体详解

22.1 <img> 关键属性

  • alt：替代文本（必须）
  • width/height：避免布局抖动
  • loading="lazy"：懒加载

22.2 <video> / <audio> 属性

  • controls：显示控制条
  • autoplay：自动播放（常需配合 muted）
  • loop：循环
  • muted：静音
  • playsinline：移动端内联播放
  • preload：none / metadata / auto
  • poster：封面图（video）

22.3 <track> 字幕

  • kind：subtitles / captions
  • srclang：语言
  • label：显示名称

23. 资源加载与性能提示

23.1 <link> 常见 rel

  • stylesheet：CSS
  • preload：预加载关键资源
  • prefetch：空闲加载
  • preconnect：提前建立连接
  • dns-prefetch：提前解析 DNS
  • icon：网站图标

23.2 <script> 加载策略

  • 默认阻塞解析
  • defer：并行下载，解析完执行
  • async：下载完成即执行
  • type="module"：ESM 模块

24. 表格可访问性

  • caption 提供表格标题
  • th 使用 scope（col/row）
  • 复杂表格可使用 headers 与单元格 id 对应

25. SEO 基础

  • 结构清晰、标题层级正确
  • title 与 meta description 唯一且准确
  • 重要内容避免全部由脚本生成
  • 为图片提供 alt

26. 安全与隐私提示

  • target="_blank" 搭配 rel="noopener"
  • iframe 可用 sandbox 限制权限
  • 使用 referrerpolicy 控制 Referer

27. 已废弃或不推荐使用

  • 结构类：center、font、big
  • 表现类：bgcolor、align 等旧属性
  • 用 CSS 替代表现控制

28. 编写规范建议

  • 始终包含 <!doctype html>
  • meta charset 放在 head 最前面
  • 为 html 设置正确 lang
  • 标签与属性小写更统一
  • 结构清晰，语义优先

29. 空元素与可省略结束标签

29.1 空元素（Void Elements）

不需要结束标签，不能包含内容：

  • area、base、br、col、embed、hr、img、input
  • link、meta、param、source、track、wbr

29.2 可省略结束标签（常见）

浏览器可自动补全结束标签，但建议明确书写：

  • li、dt、dd、p、rt、rp、optgroup、option
  • thead、tbody、tfoot、tr、th、td

30. HTML 实体与字符编码

30.1 常见实体

  • &lt; 表示 <
  • &gt; 表示 >
  • &amp; 表示 &
  • &quot; 表示 "
  • &nbsp; 不换行空格

30.2 编码建议

  • 使用 UTF-8（<meta charset="utf-8">）
  • 避免在不一致编码环境下混用中文与特殊符号

31. 元素分类详解（按功能）

31.1 文档与元数据

  • html：根元素
  • head：元数据容器
  • title：页面标题
  • meta：元信息
  • link：外部资源
  • base：基准 URL
  • style：内联样式

31.2 语义结构

  • header：页眉或区块头
  • nav：导航区域
  • main：主体内容
  • section：主题分区
  • article：独立内容
  • aside：侧栏或附属内容
  • footer：页脚

31.3 文本与段落

  • p：段落
  • h1-h6：标题
  • blockquote：块级引用
  • q：行内引用
  • address：联系信息

31.4 文本级语义

  • strong、em：强调
  • b、i、u：表现性文本
  • small：次要信息
  • mark：高亮
  • time：时间
  • abbr：缩写
  • cite：作品名
  • dfn：定义

31.5 编辑与标注

  • ins：插入内容
  • del：删除内容
  • s：不再准确/不再相关

31.6 代码与技术内容

  • code：代码
  • pre：预格式文本
  • kbd：键盘输入
  • samp：程序输出
  • var：变量
  • sub、sup：上下标

31.7 列表

  • ul、ol、li
  • dl、dt、dd

31.8 嵌入内容

  • img：图像
  • audio、video：音视频
  • source：媒体源
  • track：字幕
  • iframe：嵌入页面
  • embed、object：外部对象
  • canvas：画布
  • svg：矢量图

31.9 交互与披露

  • a：链接
  • button：按钮
  • details + summary：折叠面板
  • dialog：对话框

31.10 脚本与模板

  • script：脚本
  • noscript：禁用脚本提示
  • template：模板片段

31.11 表格

  • table、caption、thead、tbody、tfoot
  • tr、th、td、colgroup、col

31.12 表单

  • form、input、textarea、select、option、optgroup
  • label、fieldset、legend
  • button、datalist、output、progress、meter

32. 重要属性清单（补充）

32.1 <a>

  • href、target、rel、download、hreflang

32.2 <img>

  • src、alt、width、height、loading、decoding

32.3 <link>

  • rel、href、as、media、crossorigin

32.4 <meta>

  • charset
  • name + content（description、viewport、robots）
  • http-equiv（兼容旧行为）

32.5 <script>

  • src、defer、async、type、nomodule

32.6 <form>

  • action、method、enctype、target、autocomplete

33. 语义化与无语义元素取舍

  • 先选语义元素，再考虑 div
  • 对内容结构有意义时优先 section/article
  • 不要用 div + span 替代表单或表格语义

34. HTML 解析与 DOM

  • 浏览器会将 HTML 解析成 DOM 树
  • 解析过程遇到错误通常会容错
  • 正确嵌套能避免异常结构

35. 常见错误与排查

  • 忘记 alt 导致可访问性问题
  • label for 与 input id 不一致
  • 资源路径错误导致 404
  • 表单 name 缺失导致无法提交值
  • target="_blank" 未加 rel="noopener"

36. HTML 验证工具

  • W3C Validator：检测语法与结构问题
  • Lighthouse：性能、可访问性、SEO
  • 浏览器 DevTools：结构与样式排查
